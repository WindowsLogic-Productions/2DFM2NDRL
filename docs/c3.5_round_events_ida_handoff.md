## Target function

| | |
|---|---|
| Symbol | `vs_round_function` |
| Address | `0x004086A0` |
| Size | `0x1251` |
| Returns | `char` |
| Args | `()` |
| Caller | `update_game_object @ 0x0040C130` (function-table dispatch — DO NOT detour the caller, it's generic) |

`vs_round_function` is the central round state machine for FM2K's vs/story/team
modes. It runs once per frame per "round-state object" (object-pool entry of
type `1`). The function dispatches on a substate at `obj+0x152` (=338) and
mutates that field as it advances the machine.

## Substate field

```c
// the project already uses this exact access pattern at hooks.cpp:686-689
void* slot = *(void**)0x4CFA00;          // g_object_data_ptr — global pointer-to-current-slot
int   substate = *(int*)((char*)slot + 0x152);   // 0x152 = 338 decimal
```

Field is a 4-byte enum. The values relevant to us:

| Value | Symbol | Where in the function |
|---|---|---|
| `100` (`0x64`) | `RSS_BATTLE_INIT` | case body at `0x408970` — spawns fighters, resets HP/timer, begins round |
| `300` (`0x12C`) | `RSS_OUTCOME_DECIDE` | case body at `0x408F68` — finalize HP%, route to win banner |
| `900` (`0x384`) | `RSS_ROUND_END_BANNER` | case body at `0x4096CE` — round fully resolved, banner about to spawn |

Full enum is in IDA (`RoundSubstate`); only the three above matter for C3.5.

## ROUND_START emit edge

Detect the post-call edge `pre == 100 && post == 101`.

**Why this edge:** `RSS_BATTLE_INIT` body at `0x408970` runs once at the start
of every round (fires for round 1 from `RSS_INIT_LOAD_STAGE`, fires for
rounds 2+ from `vs_round_next_round_reset @ 0x4097F0`). After running, the body
sets `unknown_152 = 101` (`RSS_ROUND_ANNOUNCE_WAIT`). This is the only
transition `100 → 101` in the function — exactly one edge per round.

State available at that edge (all populated by `RSS_BATTLE_INIT` body before
return):

| Payload field | Source address | Type | Notes |
|---|---|---|---|
| `round_idx` | hook-side counter | `u8` | Initialize on `Netplay_StartBattle`, increment in this hook. Don't derive from `round_wins` — draws bump both wins by 1 each, breaking the formula. |
| `p1_hp_max` | `0x4DFC91` (`g_charslot0_hp_max`) | `u16` | Loaded by `player_data_file_loader` before this case runs. |
| `p2_hp_max` | `0x4EDCD0` (`g_charslot1_hp_max`) | `u16` | Stride from P1 = `0xE03F` (57407). |
| `timer_seconds` | `(g_score_value + 1) / 100` from `0x470050` | `u16` | `RSS_BATTLE_INIT` sets `g_score_value = 100 * stage_timer - 1`. Add 1, divide by 100 to recover seconds. |

## ROUND_END emit edge

Detect the post-call edge `pre != 900 && post == 900`.

**Why this edge:** Multiple converging paths set `unknown_152 = RSS_ROUND_END_BANNER`:

- `vs_round_win_tail_tick` (countdown reaches 0 after P1_WIN/P2_WIN/DRAW/DOUBLE_KO/PERFECT_KO sequences)
- `0x4095D9` (RSS_P2_PERFECT_KO_TAIL non-perfect path)
- `0x40950A` (RSS_P1_PERFECT_KO_TAIL non-perfect path)

All of these produce a single `* → 900` edge per round. By that frame the
`result_kind` globals, `round_wins`, and HP have all been written by the
prior win-banner case bodies. ✓ verified single-edge by tracing every write
to `unknown_152` in the function.

| Payload field | Source address | Type | Derivation |
|---|---|---|---|
| `winner_idx` | `0x4DFD87` / `0x4EDDC6` | `u8` | `result_kind == 1` → win, `2` → draw, `3` → loss. Decode: `r1==1 ? 0 : (r2==1 ? 1 : 2)` (0=P1, 1=P2, 2=draw). |
| `p1_hp_remaining` | `0x4DFC85` (`g_charslot0_hp`) | `u16` | Project already uses `HP_BASE = 0x4DFC85` at hooks.cpp:650. |
| `p2_hp_remaining` | `0x4EDCC4` (`g_charslot1_hp`) | `u16` | `HP_BASE + 0xE03F`. |
| `frames_elapsed` | hook-side delta | `u32` | Snapshot a frame counter at ROUND_START, subtract here. `g_score_value` is countdown-style and gets clamped at 0, not safe for delta. |

## Recommended hook (sketch)

MinHook detour on the function. The project already has MinHook plumbing for
similar small-surface hooks. Do NOT detour `update_game_object` — it's a
generic table dispatch and would force you to filter by `g_object_data_ptr`
type each call.

```cpp
typedef char (__cdecl *VsRoundFunc_t)();
static VsRoundFunc_t orig_vs_round_function = nullptr;

// Reset on Netplay_StartBattle, snapshot on ROUND_START
static uint8_t  s_round_idx_counter = 0;
static uint32_t s_round_start_frame = 0;

static char __cdecl Hook_vs_round_function() {
    void* slot_pre = *(void**)0x4CFA00;
    int pre = slot_pre ? *(int*)((char*)slot_pre + 0x152) : -1;

    char ret = orig_vs_round_function();

    void* slot_post = *(void**)0x4CFA00;
    int post = slot_post ? *(int*)((char*)slot_post + 0x152) : -1;

    // Only emit on the host, never during a rollback re-sim
    if (Netplay_IsHost() && !g_is_rolling_back) {
        // ROUND_START — RSS_BATTLE_INIT body just ran
        if (pre == 100 && post == 101) {
            s_round_idx_counter++;
            s_round_start_frame = g_current_frame_idx;

            uint16_t p1_hp_max     = (uint16_t)*(uint32_t*)0x4DFC91;
            uint16_t p2_hp_max     = (uint16_t)*(uint32_t*)0x4EDCD0;
            int32_t  score         = *(int32_t*)0x470050;
            uint16_t timer_seconds = (uint16_t)((score + 1) / 100);

            SpectatorNode_AppendRoundStart(
                s_round_idx_counter, p1_hp_max, p2_hp_max, timer_seconds);
        }

        // ROUND_END — substate just transitioned to RSS_ROUND_END_BANNER
        if (pre != 900 && post == 900) {
            uint32_t r1_kind = *(uint32_t*)0x4DFD87;
            uint32_t r2_kind = *(uint32_t*)0x4EDDC6;
            uint8_t winner_idx = (r1_kind == 1) ? 0
                              : (r2_kind == 1) ? 1
                              : 2;
            uint16_t p1_hp = (uint16_t)*(uint32_t*)0x4DFC85;
            uint16_t p2_hp = (uint16_t)*(uint32_t*)0x4EDCC4;
            uint32_t frames_elapsed = g_current_frame_idx - s_round_start_frame;

            SpectatorNode_AppendRoundEnd(
                winner_idx, p1_hp, p2_hp, frames_elapsed);
        }
    }

    return ret;
}
```

`Netplay_StartBattle` resets `s_round_idx_counter = 0` (so round 1 inside
each match emits idx=1). `Netplay_EndBattle` doesn't need to touch it —
the next StartBattle resets.

## Determinism note

Both edges fire from frame-deterministic sites: every call to
`vs_round_function` happens via `update_game_object` inside the main game
loop, which is already gated by GekkoNet's frame-advance. The pre/post
substate snapshots run on the same frame as the original call — the spectator
walking the replay sees the same edges at the same input-frame index when
the host's `OnFrameConfirmed` posts the corresponding INPUT event.

The host emits ROUND_START / ROUND_END as ordered ops in the same
`session_events` queue that carries INPUTs (via the existing `Append*`
helpers in `spectator_node.cpp`). On the spectator side they apply at
queue-head drain just like the other non-INPUT ops — no new ingress path
needed.

## Out of scope for the implementer

- FM95 emit sites (different binary, separate hand-off).
- `MATCH_END` payload extension (winner_idx + rounds_won_p1/p2). Mentioned in
  the recommendation chain but a separate small-surface change in
  `Netplay_EndBattle` — not part of C3.5 itself.
- `round_offsets[8]` header pass on the writer side. That's C7, separate
  commit; the round events landing here unblock it but the writer agent owns
  it.

## Open questions for the implementer

1. **Where does `s_round_idx_counter` live?** Suggest a static in
   `spectator_node.cpp` next to the existing per-match counters, exposed via
   `SpectatorNode_OnBattleStart()` reset hook.
2. **`g_current_frame_idx` source?** The existing
   `SpectatorNode_OnFrameConfirmed` already takes a frame counter; reuse
   `g_state.total_input_count` from that path for `s_round_start_frame`.
3. **Spectator-side payload application?** Per the C3.5 design, ROUND_START /
   ROUND_END are informational on the spectator — no engine memory writes
   needed (banner spawning is driven by the simulation, not by these events).
   Just dequeue + log, plus optionally feed an overlay. The `.fm2krep` writer
   (C7) is the consumer that materializes them as `round_offsets[]`.
