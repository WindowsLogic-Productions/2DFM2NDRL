## Launcher (FM2K_RollbackLauncher.exe)

- 🔴 **Pull registry from server.** `kKnownExes` is hardcoded; should fetch from `stats.2dfm.org/api/games`. Adding a game in `/admin/` should surface in the launcher next start.
- 🔴 **Crash/exception log auto-upload.** *(see "Crash log upload" section below — own design)*
- 🟡 **In-game lobby UI / chat.** Transport built (per `project_chat_ui_plan.md`), DirectX overlay hook documented, not shipped. Long-term direction is "auto-proxy DLL so the game is the lobby client" (`project_in_game_lobby_direction.md`).
- 🟡 **Auto-claim `held_by[]` on first launch.** Launcher knows the user has a game (matched its xxh64 against registry). One POST to `/admin/api/g/{id}/held/toggle` on first run flags ownership without manual editor intervention. Requires the user to have a hub_token (already have via Discord OAuth pairing).
- 🟡 **In-launcher auto-update.** Distribution is currently zips in `dist/`. Need a check-on-start that fetches latest tag and prompts.
- ⚪ Replay viewer, per-game keybind config, patron-status indicator.

## Hub (server-side, hub.py + bot)

- 🔴 **Health endpoint + monitoring.** No way to know hub is up without SSH. Cheapest fix: `/health` returning `{uptime, lobbies, users}`, cron pinger that posts a Discord webhook on failure.
- 🟡 **ELO / ranking.** W/L/D is tracked per character + pilot in `aggregate_user`/`aggregate_game`; rating system would unlock public leaderboards.
- 🟡 **Match disputes UI.** `match_disputes.json` exists; nothing surfaces it.
- 🟡 **Replay upload/storage.** Biggest content-shaped hole. We log matches but don't keep replay bytes. Needs design — file size, retention, public access policy.
- ⚪ Tournament brackets, persistent player profiles, scheduled events.

## Public stats (stats.2dfm.org)

- 🔴 **Drop Caddy basic_auth.** The Caddyfile comment literally says "Flip off once we go public." That day is here — admin editor is live, hub is recording matches, character winrates work. Editor at `/admin/` is properly auth-gated; the public read pages can be open.
- 🟡 **Render `banner_url` / `thumb_url`.** Fields are editable, never displayed on `/g/{id}`. Cheap visual identity win.
- 🟡 **Per-editor activity dashboard.** `editlog.jsonl` has all the data; no UI.
- 🟡 **Public versioned API.** `/api/v1/matches` exists, nothing for games / characters / leaderboards.
- ⚪ Time-series (matches/day), "epic matches" board, replay download.

## Admin editor / wiki (/admin/)

- 🟡 **Markdown rendering for `description`.** Currently plain text; want links + bold + paragraphs at minimum. Use a server-side renderer (e.g. `markdown` package) so output is deterministic and CSP-safe.
- 🟡 **Recent activity / per-game edit timeline.** editlog has the data; no UI yet. Useful for review + new-editor onboarding.
- 🟡 **Image upload.** Banner / thumb / character art are URL-only. Need server-side upload + storage + size cap.
- 🟡 **Rename `game_id` flow.** 268 imported atwiki games have `atwiki-N` placeholder slugs. `game_id` is intentionally NOT in `_ALLOWED_PATCH_FIELDS` because changing it orphans match history; need a dedicated rename endpoint that updates `aliased_ids[]` so old `game_id` keeps resolving.
- ⚪ Bulk edit, drafts/proposals, public `/wiki/g/{id}` (read-only public counterpart of the editor).
- ⚪ Mobile editor (explicitly scrapped earlier — revisit later).

## Cross-cutting / ops

- 🔴 **`admin_tokens.json` has no offsite copy.** Repo correctly gitignores it; needs separate encrypted store. `age` + a single-purpose private repo would take ~30 min.
- 🟡 **109 NEEDS_HUNT games still need manual research.** No tracking of who's looked at what — editors duplicate work. Add a "claimed for research" flag separate from `held_by[]`.
- 🟡 **Discord notifications.** Bot exists (`fm2k-bot.service`). Could push: lobby-formed, match-finished, new-editor-claims-a-game, NEEDS_HUNT recovery.
- ⚪ Staging environment, deployment automation (currently manual rsync), centralized log aggregation, secrets vault.

## Crash log upload — design draft

**Problem:** users hit a crash, by the time anyone asks for logs they've already started another session and `c:/games/2dfm/{game}/logs/FM2K_P1_Debug.log` is overwritten. Repeat. We strip PII already, so we can collect aggressively.

**Proposed flow**

1. **Quill log sink change** — keep two files: the current `FM2K_P*_Debug.log` (overwritten per session) AND a `FM2K_session_<unix_ts>_<pid>.log` (immutable per session). On clean exit, the session log is moved to a `logs/sent/` directory; on crash, it stays in `logs/pending/`.

2. **SetUnhandledExceptionFilter / minidump hook** in the launcher and FM2KHook.dll — when a hard crash fires, write a minidump alongside the session log and flag the session as crashed (e.g. `logs/pending/<session>.crashed` marker file).

3. **Boot-time scan** — on launcher start, walk `logs/pending/`. For any session whose marker indicates crash (or that's older than N hours and was never moved to `sent/`), upload via:
   ```
   POST hub.2dfm.org/logs/upload
     headers: Authorization: Bearer <hub_token>
     multipart:
       session_id      = unix_ts + pid
       hook_version    = (e.g. v0.2.34)
       launcher_version
       game_id         = (e.g. pkmncc)
       log_file        = the session log
       minidump        = optional .dmp blob
   ```

4. **Hub endpoint** writes to `/home/fm2k/hub/crash_logs/<YYYY-MM-DD>/<session>/` and immediately notifies a Discord webhook (channel: dev-only) with the user nick + game + first few lines of stack trace.

5. **Retention** — auto-delete after 30 days unless flagged for keep. Bound by a global cap (e.g. 1 GB) so a misbehaving build can't fill the disk.

**PII guarantees** — quill sinks already strip user paths and Discord IDs. Need to re-audit before the endpoint goes live. Include a launcher-side opt-out (env var or settings.json flag) for users who really don't want to send.

**RPO** — within seconds of next launcher start. Crashed sessions never overwrite each other (unique session id).

## My pick for next-up, in order

1. **Drop Caddy basic_auth on stats** (5 min) — site is ready, gating is friction.
2. **Rename `game_id` flow** (1 hour) — clean up 268 placeholder slugs, preserve match history via `aliased_ids[]`.

4. **Hub `/health` + Discord webhook on failure** (1 hour) — cheap monitoring insurance.
5. **`admin_tokens.json` encrypted offsite** (30 min) — close the data-loss risk.
6. **Crash log upload** (half-day to design + ship the launcher half) — kills the "what was your crash?" conversation forever.
7. **Public `/wiki/g/{id}` + markdown for description** (2 hours) — turns game page into a visitable thing, not a stats dump.
8. **Replay upload** (half-day, own design pass) — biggest content gap.
